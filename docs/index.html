RE-Argo とは

出発点は NSA Ghidra の解析パイプラインを Rust で再実装することでした。そこから、マルウェア家族判定・能力検出・IoC 抽出の層を積み重ね、 独立した RE + トリアージ・ワークベンチ へと成長しています。GUI は持たず、 CLI とライブラリ、そして AI エージェント（MCP）から直接操作する設計です。

🚢 Argo は未知の海へ漕ぎ出した神話の船。RE-Argo は、未知のバイナリへ持ち込む計器です。RE = Reverse Engineering。

解析コア（Ghidra スタイル）

マルチアーキテクチャのリフティングと逆コンパイルを支える基盤。

🧩

6 アーキテクチャ

x86 / x64・ARM / AArch64・RISC-V・MIPS・PowerPC・SPARC。命令は 74 オペコードの P-code IR にリフトされ、フルエミュレーション対応。

⚙️

68 アナライザー

関数探索（再帰下降＋リニアスイープ）、312 エントリのシグネチャ DB、VSA、RTTI / VTable 復元、呼び出し規約推定、anti-debug / crypto / loop 検出などを優先度順に実行。

🔬

逆コンパイラ

SSA 構築・6 つの最適化パス・型推論。C / Rust 疑似コードを出力し、インライン注釈とシグネチャ対応のコールレンダリング付き。

📐

デバッグ情報

DWARF（関数 / 型 / 引数 / 行番号 / ソースプレート）、PDB ヘッダー / 型。SLEIGH ランタイムも内蔵。

📦

バイナリ形式

ELF・PE（.pdata / TLS コールバック / IAT / Rich Header）・Mach-O（ObjC クラス / メソッド / ivar）・COFF・raw。

🐞

デバッガ基盤

GDB RSP プロトコル（クライアント＋サーバ）、ブレークポイント、ウォッチポイント、syscall エミュレーション、スナップショット。MCP 統合も。

マルウェアトリアージ層

RE-Argo を際立たせる、解析コアの上に載る独自レイヤー。

🩺

ワンスクリーン・トリアージ

triage がフルパイプラインを実行し、形式 / アーキ / 識別子 / ハッシュ / パッカー / capa / CWE / IoC / タグ数を 1 枚のレポートに集約します。

🧬

家族クラスタリング

imphash、TLSH（内容ファジー）、RichHash（MSVC ツールチェイン）を info / triage で提示。tlsh-diff で類似度比較。

🎯

能力検出 & YARA-lite

Capa スタイルの 20 ルールエンジンと、YARA のストリクトサブセット（テキスト＋ワイルドカード hex、ブール条件、N of them）。

🔓

FLOSS-lite & パッカー識別

XOR / ROL / ADD の難読化文字列デコーダと、DIE / PEiD スタイルの UPX / ASPack / Themida / VMProtect ほかの識別。

🌐

IoC 抽出

URL / IPv4 / IPv6 / email / レジストリキー / named-pipe / mutex / パス / ETH / BTC / user-agent / ドメインの 13 種分類。

⚠️

CWE 脆弱性パターン

危険な API 呼び出しに CWE id（78 / 120 / 134 / 242 / 330 / 426 / 676）を付与。ROP / JOP / COP ガジェット探索、埋め込みファイル検出も。

クイックスタート

ビルドして、未知のサンプルにまず triage を当てる。

# ビルド（バイナリは target/release/re-argo に生成）
cargo build --release

# ワンスクリーンのマルウェアトリアージ — 最初に打つ一手
target/release/re-argo triage <binary>

# 名前 / シンボル / コメント / タグを横断検索
target/release/re-argo find <binary> "printf"

# アドレス指定で逆コンパイル
target/release/re-argo decompile <binary> --address 0x401000

# IoC（URL / IP / レジストリキー / …）抽出
target/release/re-argo ioc <binary>

# Capa スタイルの能力レポート
target/release/re-argo capa <binary>

全コマンド・全フラグ・出力形式・複数コマンドのレシピは ドキュメントページを参照してください。

アーキテクチャ

疎結合な 10 個の Rust クレートで構成。クレートは reargo-* 接頭辞、公開バイナリは re-argo。

re-argo (reargo-cli)
  ├── reargo-decompile (SSA, 最適化, C/Rust 出力)
  │     ├── reargo-lift (x86 / ARM / RISC-V / ... → P-code)
  │     └── reargo-sleigh (SLEIGH ランタイム)
  ├── reargo-analysis (68 アナライザー + トリアージ層)
  │     └── reargo-program (プログラムモデル + タグ)
  ├── reargo-arch (6 アーキテクチャ, .cspec / .pspec)
  ├── reargo-loader (ELF / PE / Mach-O / COFF, DWARF, PDB)
  ├── reargo-emulator (P-code エミュレーション, GDB RSP)
  └── reargo-core (アドレス, P-code IR, データ型)
